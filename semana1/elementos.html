"TAGS en HTML5"


Estructura básica de una página web:


- DOCTYPE: ("< !DOCTYPE >")

La etiqueta Doctype o “Declaración del tipo de documento” es lo primero que debe tener todo documento web compatible. Su propósito es informar al navegador el tipo de documento que está a punto de procesar.


- HTML: (< html > < /html >)

Constituye la raíz de todo documento HTML bien formado y se supone que encierre a todos los demás elementos entre sus etiquetas.
Incluye elementos como: < head >, < title >, < body >, < header >, < footer >, < article >, < section >, < p> , < div >, < span >, < img >, < aside >, < audio >, < canvas >, < datalist >, < details >, < embed >, < nav >, < output >, < progress >, < video >, < ul >, < ol >, < li > y muchos otros.


- HEAD: (< head > < /head >)

El elemento head es el contenedor de una serie de elementos que proveen metadatos (datos que describen otros datos) para el documento. Estos metadatos pueden especificar el título (elemento title), el juego de caracteres, el autor, la descripción o palabras clave del documento (elemento meta), una hoja de estilo o información relacional (elemento link) o declaraciones de estilo incrustadas (elemento style), entre otros.
La presencia del elemento title es obligatoria dentro de la sección head de un documento HTML. La única situación en la que puede ser omitido es cuando el título es provisto en otro lugar (por ejemplo, por el elemento padre de un iframe).


- TITLE: (< title > < /title >)

El elemento title provee un título o nombre para el documento. Este título debe proporcionar una corta, pero precisa descripción del contenido del documento o su propósito aun cuando es sacada de contexto (como, por ejemplo, cuando se muestra en los resultados de una búsqueda).


- BODY: (< body > < /body >)

El elemento body es el contenedor para todos los aspectos representables de un documento. Este es el lugar donde los autores arrojan todos los elementos y contenidos que sus visitantes alcanzarán. El elemento body es parte de la estructura básica de un documento.
Atributos generales:

  • alink: Color del texto de los hipervínculos cuando se selecciona.
  • background: URI de una imagen para usar como fondo.
  • bgcolor: Color de fondo del documento.
  • bottommargin: El margen de la parte inferior del < body >.
  • leftmargin: El margen de la izquierda del < body >.
  • link: Color del texto de los enlaces de hipertexto no visitados.
  • rightmargin: El margen de la derecha del < body >.
  • text: Color de primer plano del texto.
  • topmargin: El margen de la parte superior del < body >.
  • vlink: Color del texto de los enlaces de hipertexto visitados.


- H1: (< h1 > < /h1 >)

El elemento h1 representa un encabezado de nivel 1. Los encabezados (desde h1 a h6) son títulos que definen secciones implícitas en el documento y organizan el contenido en una estructura jerárquica.
Cuando los encabezados son combinados con el elemento section las secciones del documento dejan de ser implícitas para estar exactamente definidas. En tales casos, el encabezado de una sección debería preceder a todos los demás elementos de esta.


- P: (< p > y < /p >, la de cierre es opcional).

El elemento p representa un párrafo. Se considera párrafo a un bloque de texto compuesto de una o más oraciones que se refieren a un tema en particular y que normalmente se separa de otros por un espacio en blanco.
Atributos generales:

  • title: Título consultivo del elemento.
  • style: Información de estilo en línea.
  • id: Identificador único a nivel de documento.
  • class: Identificador a nivel de documento.
  • dir: Dirección del texto débil/neutral.
  • lang: Información sobre el idioma.
Atributos específicos:
  • align ("left", "center", "right", o "justify"): Indica la alineación horizontal con relación al contexto.
