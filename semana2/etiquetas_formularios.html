Etiquetas para formularios:


Tipos de datos admitidos por las etiquetas de html input, select, textarea y radio:

Etiqueta: Descripción: Ejemplo de uso:
<input> Se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario.

Escribe algo:

<input type="button"> Se usa para crear un botón en el que se puede hacer click sin ningún valor por defecto, es decir, no tiene un comportamiento predeterminado.
<input type="checkbox"> Elemento de entrada que te permite insertar un vector o array de valores.
<input type="color"> Elemento de interfaz de usuario que permite a los usuarios especificar un color, bien mediante una interfaz visual de selección, bien a través de un cuadro de texto donde ingresar un valor hexadecimal en el formato «#rrggbb».

Elige un color:

<input type="date"> Crea campos de entrada que permiten a los usuarios proporcionar una fecha, bien a través de un cuadro de texto que valida automáticamente el contenido, o bien mediante una interfaz especial de selección de fechas. El valor resultante incluye el año, el mes y el día, pero no la hora.
<input type="datetime-local"> Crea controles de entrada que permitan al usuario ingresar fácilmente tanto una fecha como una hora, incluido el año, mes y día, así como la hora en horas y minutos.
<input type="email"> Se utilizan para permitir que el usuario ingrese y edite una dirección de correo o, si se especifica el atributo multiple, una lista de direcciones de correo.
<input type="file"> Permite que el usuario elija uno o más archivos del almacenamiento de su dispositivo. Una vez elegidos, los archivos pueden cargarse en un servidor mediante el envío de formularios o manipularse mediante código JavaScript y la API de archivos.
<input type="hidden"> Permite que los desarrolladores web incluyan datos que los usuarios no pueden ver ni modificar cuando se envía un formulario. Por ejemplo, el ID del contenido que se está solicitando o editando actualmente, o un token de seguridad único. Las entradas ocultas son completamente invisibles en la página renderizada y no hay forma de hacerlas visibles en el contenido de la página.

<input type="image"> Se utilizan para crear botones gráficos de envío, es decir, botones de envío que toman la forma de una imagen en lugar de texto.
<input type="month"> Crea campos de entrada que permiten al usuario ingresar un mes y un año, lo que permite ingresar fácilmente un mes y un año. El valor es una cadena cuyo valor tiene el formato "AAAA-MM", donde AAAA es el año de cuatro dígitos y MM es el número del mes.
<input type="number"> Usados para permitir al usuario ingresar un número. Éstos incluyen validación incorporada para rechazar entradas no numéricas. El navegador puede optar por proveer flechas de pasos para permitir al usuario incrementar y decrementar el valor usando su ratón o simplemente pulsando con la punta del dedo.
<input type="password"> Proporciona una forma para que el usuario ingrese una contraseña de forma segura. El elemento se presenta como un control de editor de texto, sin formato, de una línea, en el que el texto está oculto para que no pueda leerse, generalmente reemplazando cada carácter con un símbolo como el asterisco ("*") o un punto ("•").

Ingresa contraseña:
<input type="radio"> Generalmente se utiliza en grupos de radio: colecciones de botones de radio que describen un conjunto de opciones relacionadas. Solo se puede seleccionar un botón de opción en un grupo determinado al mismo tiempo. Los botones de opción se representan normalmente como círculos pequeños, que se rellenan o resaltan cuando se seleccionan.

<input type="range"> Permite que el usuario especifique un valor numérico comprendido entre un valor mínimo y máximo. El valor exacto, sin embargo, no se considera importante. Se repesenta típicamente como un "tirador" o un control deslizante en lugar de un campo de texto como otros tipos de input.
<input type="reset"> Se representan como botones, con un controlador de eventos de clic predeterminado que restablece todas las entradas en el formulario a sus valores iniciales.

<input type="search"> Son campos de texto diseñados para que el usuario ingrese consultas de búsqueda. Estos son funcionalmente idénticos a las entradas de texto, pero el agente de usuario puede aplicarles un estilo diferente.
<input type="submit"> Se representan como botones. Cuando ocurre el evento de clic (generalmente porque el usuario hizo clic en el botón), el agente de usuario intenta enviar el formulario al servidor.
<input type="tel"> Se utilizan para permitir que el usuario introduzca y edite un número de teléfono. A diferencia de <input type = "email"> y <input type = "url">, el valor de entrada no se valida automáticamente a un formato en particular antes de que se pueda enviar el formulario, porque los formatos de los números de teléfono varían mucho en todo el mundo.
Formato: 123-456-7890
<input type="text"> Crea campos de texto básicos de una sola línea.
<input type="time"> Crea campos de entrada diseñados para permitirle al usuario ingresar fácilmente una hora (horas y minutos, y opcionalmente segundos).
Horario de atención: 9am-6pm
<input type="url"> Se utilizan para permitir al usuario ingresar y editar una URL.
<input type="week"> Crea campos de entrada que permitan la entrada fácil de un año más el número de semana durante ese año (es decir, semana 1 a 52 o 53).
<select> Representa un control que muestra un menú de opciones. Las opciones contenidas en el menú son representadas por elementos <option>, los cuales pueden ser agrupados por elementos <optgroup>. La opcion puede estar preseleccionada por el usuario.

<select number> atributo size El atributo de tamaño especifica el número de opciones visibles en una lista desplegable. Si el valor del atributo de tamaño es mayor que 1, pero menor que el número total de opciones en la lista, el navegador agregará una barra de desplazamiento para indicar que hay más opciones para ver.

<select required> atributo El atributo requerido es un atributo booleano. Cuando está presente, especifica que el usuario debe seleccionar un valor antes de enviar el formulario.

<select name = "text"> atributo El atributo de nombre especifica el nombre de una lista desplegable. El atributo de nombre se utiliza para hacer referencia a elementos en un JavaScript o para hacer referencia a datos de formulario después de que se envía un formulario.

<select multiple> atributo Cuando está presente, especifica que se pueden seleccionar varias opciones a la vez. La selección de varias opciones varía en diferentes sistemas operativos y navegadores:
  • Windows: mantenga presionado el botón de control (ctrl) para seleccionar varias opciones

  • Mac: mantenga presionado el botón de comando para seleccionar múltiples opciones

    • Debido a las diferentes formas de hacer esto, y debido a que tiene que informar al usuario que la selección múltiple está disponible, es más fácil usar casillas de verificación.

    <select form = "form_id "> atributo El atributo de formulario especifica el formulario al que pertenece la lista desplegable. El valor de este atributo debe ser igual al atributo id de un elemento <form> en el mismo documento.
    <select disabled> atributo El atributo desactivado es un atributo booleano.
    Cuando está presente, especifica que la lista desplegable debe estar deshabilitada.
    Una lista desplegable deshabilitada es inutilizable y no se puede hacer clic en ella.
    El atributo deshabilitado se puede configurar para evitar que un usuario use la lista desplegable hasta que se cumpla alguna otra condición (como seleccionar una casilla de verificación, etc.). Luego, un JavaScript puede eliminar el valor deshabilitado y hacer que la lista desplegable sea utilizable.

    <select autofocus> atributo El atributo desactivado es un atributo booleano.
    Cuando está presente, especifica que la lista desplegable debe enfocarse automáticamente cuando se carga la página.

    <textarea> Representa un control para la edición mutilínea de texto sin formato.
    <textarea wrap = "soft/hard"> atributo El atributo de ajuste especifica cómo se ajustará el texto de un área de texto cuando se envíe en un formulario.
  • soft: el texto en el área de texto no se ajusta cuando se envía en un formulario. Esto es predeterminado.

  • hard: el texto en el área de texto se ajusta (contiene nuevas líneas) cuando se envía en un formulario. Cuando se usa "hard", se debe especificar el atributo "cols".

    <textarea rows = "number"> atributo El atributo de filas especifica la altura visible de un área de texto, en líneas.
    El tamaño de un área de texto también se puede especificar mediante las propiedades de alto y ancho de CSS.

    Este área de texto tiene 4 filas visibles, puedes cambiarlo usando el valor del atributo "rows".

    <textarea required"> atributo El atributo requerido es un atributo booleano. Cuando está presente, especifica que se requiere / debe completar un área de texto (para enviar el formulario).
    <textarea readonly> atributo El atributo de solo lectura es un atributo booleano. Cuando está presente, especifica que un área de texto debe ser de solo lectura.
    En un área de texto de solo lectura, el contenido no se puede cambiar, pero un usuario puede tabularlo, resaltarlo y copiar contenido de él.
    El atributo de solo lectura se puede configurar para evitar que un usuario use un área de texto hasta que se cumpla alguna otra condición (como seleccionar una casilla de verificación, etc.). Luego, se requiere un JavaScript para eliminar el valor de solo lectura y hacer que el área de texto sea editable.

    Solo lectura:
    <textarea placeholder = "text" >atributo El atributo de marcador de posición especifica una pequeña pista que describe el valor esperado de un área de texto. La pequeña sugerencia se muestra en el área de texto antes de que el usuario ingrese un valor.

    <textarea name = "text"> atributo El atributo de nombre especifica un nombre para un área de texto.
    El atributo de nombre se usa para hacer referencia a elementos en un JavaScript, o para hacer referencia a datos de formulario después de que se envía un formulario.
    <textarea maxlength = "number"> atributo El atributo "maxlength" especifica la longitud máxima (en caracteres) de un área de texto.
    <textarea form = "form_id"> atributo El atributo de formulario especifica el formulario al que pertenece el área de texto. El valor de este atributo debe ser igual al atributo id de un elemento <form> en el mismo documento.
    Nombre:

    <textarea disabled> atributo El atributo desactivado es un atributo booleano. Cuando está presente, especifica que el área de texto debe estar deshabilitada. Un área de texto deshabilitada no se puede utilizar y el texto no se puede seleccionar (no se puede copiar).
    El atributo deshabilitado se puede configurar para evitar que un usuario use el área de texto hasta que se cumpla alguna otra condición (como seleccionar una casilla de verificación, etc.). Entonces, un JavaScript podría eliminar el valor deshabilitado y hacer que el área de texto sea utilizable.
    <textarea name="myname" dirname="myname.dir"> atributo El atributo dirname permite el envío de la dirección del texto del área de texto.
    El valor del atributo dirname es siempre el nombre del área de texto, seguido de ".dir".

    <textarea cols = "number"> atributo El atributo cols especifica el ancho visible de un área de texto.
    El tamaño de un área de texto también se puede establecer mediante las propiedades de altura y ancho de CSS.
    <textarea autofocus> atributo El atributo de enfoque automático es un atributo booleano.
    Cuando está presente, especifica que el área de texto debe enfocarse automáticamente cuando se carga la página.
